When Saint Demetrios Greek Orthodox Church in Seattle needed to bring its ancient liturgy to modern screens, they didn't hire an AV company.

They had a parishioner who happened to be a software engineer — one who saw every challenge as an architecture problem waiting to be solved. What started as "can we livestream Sunday service?" became a full-stack church AV platform: automated scheduling, robotic cameras, self-healing kiosks, and centralized monitoring — all built from scratch, one commit at a time.

This is that story.

Chapter 1

LiveStreamScheduler

It starts with knowing when to go live.

An automated scheduling engine that bridges the church calendar with YouTube and Facebook Live. It indexes upcoming streams, matches them to calendar events by start-time proximity, and presents a web dashboard for one-click broadcast management.

  • Calendar-aware stream matching with n-gram similarity fallback
  • YouTube & Facebook Live API integration
  • Web dashboard with scheduled, recorded, & unscheduled views
  • AI-assisted sermon metadata & Shorts extraction workflows

Chapter 2

Camera Control

Every broadcast needs a camera operator. Or does it?

A browser-based PTZ camera control console for Panasonic AW-series and ONVIF cameras. Multi-camera dashboard with one-click preset recall, automated sequences with boomerang and loop modes, live MJPEG/HLS preview, and a captures gallery — all from any device with a browser.

  • Multi-camera dashboard with per-camera iframe controllers
  • Canvas-based virtual joystick for real-time PTZ
  • Automated sequences: auto, semi-auto, and boomerang modes
  • ONVIF support via custom PHP SOAP proxy with WS-Security
  • Captures gallery with multi-select zip download

Chapter 4

VideoKiosk2

A display that never sleeps.

A Raspberry Pi kiosk system that streams live video with automatic failover. VLC handles the primary stream with freeze detection and CPU monitoring; if it fails, Midori takes over with a browser-based fallback. CEC integration controls the TV power state, and a scheduler polls the calendar API for automated restarts.

  • Self-healing VLC → Midori failover chain
  • HDMI-CEC TV power control via cec-utils
  • Calendar-driven scheduled restart system
  • One-command installer generates all runtime artifacts
Bash systemd VLC Raspberry Pi CEC

Chapter 5

Church Monitoring

Because even the best systems need oversight.

A lightweight centralized monitoring system for church AV infrastructure. Clients collect system metrics via cron; the server fetches on demand with parallel queries and caching. The HTTPS dashboard shows device health, process status, CEC TV control, and on-demand screenshot capture — all secured with mutual TLS and token-based enrollment.

  • Pull-based architecture: server fetches from clients on demand
  • Mutual TLS with token-based device enrollment
  • CEC TV control: power on, standby, set active input
  • Remote screenshot capture via X11 display grab
  • Parallel client queries with 30-second response caching
Bash CGI mTLS Apache jq

Bonus Chapter

MinerManager

Why pay for gas when you can mine bitcoin?

When Seattle winters get cold, most people turn up the thermostat. This project supplements gas heating with bitcoin mining rigs — the waste heat warms the house while the miners earn cryptocurrency. MinerManager is the dashboard that keeps it all under control: real-time hashrate monitoring, temperature tracking, and remote management via SSH key-based authentication.

  • Real-time hashrate and temperature monitoring via AJAX polling
  • SSH key-based authentication for secure remote miner access
  • Dark-themed dashboard with card-based miner layout
  • Consistent number formatting across dashboard and API
