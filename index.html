When Saint Demetrios Greek Orthodox Church in Seattle needed to bring the Divine Liturgy to parishioners who couldn't be there in person, they didn't hire an AV company.

They had a parishioner who happened to be a software engineer — one who saw every challenge as an architecture problem waiting to be solved. What started as "can we livestream the Liturgy?" became a full-stack parish AV platform: automated scheduling, robotic cameras, lobby displays, and centralized monitoring — all built from scratch, one commit at a time.

This is that story.

Chapter 1

LiveStreamScheduler

It starts with knowing when to go live.

A semi-automated scheduling tool that bridges the parish calendar with YouTube and Facebook Live. It pulls upcoming services from Google Calendar, matches them to scheduled broadcasts by start-time proximity, and presents everything in a single dashboard — no more hunting through the calendar to find when the next Liturgy or Vespers is happening. After a service is recorded, AI-powered workflows generate sermon titles, descriptions, and YouTube Shorts — guided by custom skill files with extensive Orthodox theology and terminology rules so the output is always reverent and tradition-accurate.

  • Calendar-aware stream matching with n-gram similarity fallback
  • Web dashboard with scheduled, recorded, & unscheduled views
  • AI-assisted sermon metadata & Shorts extraction workflows
Python YouTube API yt-dlp HTTP Server

Chapter 2

Camera Control

Every broadcast needs a camera operator. Or does it?

A browser-based PTZ camera control console for Panasonic AW-series and ONVIF cameras. Multi-camera dashboard with one-click preset recall, automated sequences with boomerang and loop modes, live MJPEG/HLS preview, and a captures gallery — all from any device with a browser.

  • Multi-camera dashboard with per-camera iframe controllers
  • Canvas-based virtual joystick for real-time PTZ
  • Automated sequences: auto, semi-auto, and boomerang modes
  • ONVIF support via custom PHP SOAP proxy with WS-Security
  • Captures gallery with multi-select zip download
JavaScript PHP ONVIF/SOAP Apache Canvas API

Chapter 3

Church Calendar

Keeping the community informed, beautifully.

A full-screen calendar display designed to run on lobby screens and kiosks. Pulls events from Google Calendar and renders them as auto-rotating slides, with flyers and event images interspersed from local folders, Google Drive, or Google Photos — so parishioners see both what's coming up and the artwork promoting it. Five color themes match liturgical seasons, from Lenten purple to Paschal gold, and the whole thing is tuned to run on a Raspberry Pi Zero 2W.

  • Google Calendar API integration with multi-calendar support and caching
  • Image slides interspersed between events — sourced from local files, Google Drive, or Google Photos
  • Five liturgical color themes with WCAG AA contrast compliance
  • Automatic WebP conversion and responsive image optimization
  • Optimized for Raspberry Pi Zero 2W with offline-capable cached operation
Python Flask HTML/CSS Google Calendar API

Chapter 4

VideoKiosk2

A display that never sleeps.

A Raspberry Pi kiosk system that powers the TVs around the parish — in the nursery, outside the nave, anywhere parishioners gather. During services, VLC streams the live broadcast; when the Liturgy ends, the display automatically transitions to the calendar of upcoming events, making the most of screens that are already on. CEC integration controls TV power, and a scheduler polls the calendar API for automated restarts.

  • Automatic livestream → calendar transition between services
  • HDMI-CEC TV power control via cec-utils
  • Calendar-driven scheduled restart system
  • One-command installer generates all runtime artifacts
Bash systemd VLC Raspberry Pi CEC

Chapter 5

Church Monitoring

Because even the best systems need oversight.

A lightweight centralized monitoring system for church AV infrastructure. Clients collect system metrics via cron; the server fetches on demand with parallel queries and caching. The HTTPS dashboard shows device health, process status, CEC TV control, and on-demand screenshot capture — all secured with mutual TLS and token-based enrollment.

  • Pull-based architecture: server fetches from clients on demand
  • Mutual TLS with token-based device enrollment
  • CEC TV control: power on, standby, set active input
  • Remote screenshot capture via X11 display grab
  • Parallel client queries with 30-second response caching
Bash CGI mTLS Apache jq

Bonus Chapter

MinerManager

Why pay for gas when you can mine bitcoin?

When Seattle winters get cold, most people turn up the thermostat. This project supplements gas heating with bitcoin mining rigs — the waste heat warms the house while the miners earn cryptocurrency. MinerManager is the dashboard that keeps it all under control: real-time hashrate monitoring and remote management via SSH key-based authentication.

  • Real-time hashrate monitoring via AJAX polling
  • SSH key-based authentication for secure remote miner access
  • Dark-themed dashboard with card-based miner layout
  • Consistent number formatting across dashboard and API
